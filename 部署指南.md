# 代码提交和部署指南

## 第一步：提交代码到 Git

### 1. 查看修改内容
```bash
git status
git diff
```

### 2. 添加所有修改的文件
```bash
git add .
```

### 3. 创建提交
```bash
git commit -m "$(cat <<'EOF'
新增：安全增强和管理功能

1. 密码安全：前端SHA256传输 + 后端bcrypt加盐哈希存储
2. 防护机制：匹配请求频率限制（10次/小时）+ 联系方式查看日志
3. 管理后台：用户管理、数据统计、账号删除功能
4. 用户权限：支持自主注销账号及相关数据
5. 功能优化：晚餐时间增加19:00-20:00时段
6. 用户体验：完善免责声明，增加详细条款

Co-Authored-By: Claude Opus 4.6 <noreply@anthropic.com>
EOF
)"
```

### 4. 推送到远程仓库
```bash
git push origin main
```

## 第二步：部署到 Render

根据你的 CLAUDE.md，你使用的是 Render 平台部署。

### 自动部署（推荐）

如果你已经配置了 Render 的自动部署：

1. **推送代码后自动触发**
   - 代码推送到 GitHub 后，Render 会自动检测
   - 自动执行构建和部署流程
   - 大约需要 3-5 分钟

2. **查看部署状态**
   - 登录 Render Dashboard: https://dashboard.render.com/
   - 找到你的服务
   - 查看 "Events" 标签，确认部署状态
   - 查看 "Logs" 标签，检查是否有错误

### 手动部署

如果需要手动触发部署：

1. 登录 Render Dashboard
2. 找到你的服务
3. 点击右上角的 "Manual Deploy" 按钮
4. 选择 "Deploy latest commit"

## 常见部署问题

### 问题1：构建失败
**原因：** 依赖安装失败或构建脚本错误
**解决：**
- 查看 Render 的构建日志
- 确认 package.json 中的脚本正确
- 检查 Node.js 版本是否兼容

### 问题2：服务启动失败
**原因：** 环境变量未配置或数据库连接失败
**解决：**
- 在 Render Dashboard 检查环境变量
- 确认 MONGODB_URI 和 JWT_SECRET 已设置
- 查看服务日志确认具体错误

### 问题3：前端页面空白
**原因：** 前端构建失败或路由配置问题
**解决：**
- 检查构建日志中的前端构建步骤
- 确认 frontend/dist 目录已生成
- 检查 server.js 中的静态文件配置

### 问题4：API 请求 404
**原因：** 路由配置错误
**解决：**
- 确认 server.js 中所有路由都已注册
- 检查 API 路径是否正确（应该是 /api/xxx）

## 回滚方案

如果部署后发现严重问题，可以快速回滚：

### 方法1：在 Render 回滚
1. 进入 Render Dashboard
2. 找到你的服务
3. 在 "Events" 中找到上一个成功的部署
4. 点击 "Rollback to this deploy"

### 方法2：Git 回滚
```bash
# 查看提交历史
git log --oneline

# 回滚到上一个提交
git revert HEAD

# 推送回滚
git push origin main
```
