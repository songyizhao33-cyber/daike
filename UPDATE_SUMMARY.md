# 更新摘要 (2026-03-06)

## 新增功能

### 1. 移动端响应式优化 ✅
- 所有页面现在都支持移动端显示
- 优化了表格、表单、按钮在小屏幕上的显示
- 课程表选择器在手机上更易操作
- 登录/注册页面自适应屏幕宽度

### 2. 校区选择功能 ✅
**代课者（管理空闲时间）：**
- 可以多选校区：邯郸、枫林、江湾、张江
- 必选项，至少选择一个校区

**需求者（发布代课需求）：**
- 单选校区
- 必选项
- 匹配时只会显示包含该校区的代课者

### 3. 频率类型选择 ✅
三种类型：
- **长期**：长期有效的时间安排
- **短期**：短期内的时间安排
- **单次**：一次性的时间安排

两个角色都需要选择频率类型（必选）

### 4. 约饭功能 ✅
全新的独立模块，包含：

**发布约饭：**
- 选择日期（不能选择过去的日期）
- 选择用餐类型：早餐/午餐/晚餐
- 选择具体时间：
  - 早餐：07:00, 07:30, 08:00
  - 午餐：11:00, 11:30, 12:00, 12:30
  - 晚餐：17:00, 17:30, 18:00, 18:30
- 选择校区（单选）
- 输入地点（如：食堂一楼）
- 可选备注

**浏览约饭：**
- 查看所有有效的约饭信息
- 可按日期和校区筛选
- 显示发布者的联系方式（微信、邮箱）
- 方便用户直接联系

**我的约饭：**
- 查看自己发布的约饭
- 可以取消约饭

## 部署说明

### 数据库迁移
由于数据模型有重大更新，建议：
1. 备份现有数据
2. 清空 `availabilities` 和 `matchrequests` 集合
3. 或者手动为现有记录添加新字段的默认值

### Render 部署
使用现有的部署配置即可，无需修改：
```bash
# Build Command
cd frontend && npm install && npm run build && cd ../backend && npm install

# Start Command
cd backend && npm start
```

### 环境变量
无需新增环境变量，使用现有配置：
- `PORT`
- `MONGODB_URI`
- `JWT_SECRET`

## 用户使用流程

### 代课匹配流程（已优化）
1. 代课者：管理空闲时间 → 选择校区（多选）→ 选择频率 → 选择时间段 → 保存
2. 需求者：发布需求 → 选择校区（单选）→ 选择频率 → 选择时间段 → 填写课程信息 → 搜索
3. 系统自动匹配符合条件的代课者
4. 需求者选择合适的代课者并通过微信/邮箱联系

### 约饭流程（全新）
1. 用户A：约饭功能 → 发布约饭 → 选择日期/时间/校区/地点 → 发布
2. 用户B：约饭功能 → 浏览约饭 → 查看约饭信息 → 查看联系方式
3. 用户B通过微信/邮箱联系用户A
4. 完成约饭
