# 快速启动指南

## 解决 MongoDB 连接问题

您遇到的连接错误是因为网络问题或 MongoDB Atlas 配置问题。有两种解决方案：

### 方案一：通过 API 初始化（推荐）

1. 先启动后端服务器：
```bash
cd backend
npm run dev
```

2. 在浏览器中访问应用并登录管理员账号

3. 使用 Postman 或浏览器开发者工具，发送请求：
```
POST http://localhost:3000/api/tarot/initialize
Headers:
  Authorization: Bearer <你的管理员token>
```

### 方案二：检查 MongoDB Atlas 配置

1. 登录 [MongoDB Atlas](https://cloud.mongodb.com/)
2. 进入 Network Access
3. 添加 IP 地址：`0.0.0.0/0`（允许所有IP访问）
4. 等待几分钟让配置生效
5. 重新运行初始化脚本

### 方案三：使用本地 MongoDB（开发环境）

如果您有本地 MongoDB：

1. 修改 `.env` 文件：
```
MONGODB_URI=mongodb://localhost:27017/substitute-matching
```

2. 启动本地 MongoDB 服务

3. 运行初始化脚本

## 本地开发启动

### 1. 安装依赖

```bash
# 后端
cd backend
npm install

# 前端
cd ../frontend
npm install
```

### 2. 配置环境变量

确保 `backend/.env` 文件存在并配置正确：
```
PORT=3000
MONGODB_URI=你的MongoDB连接字符串
JWT_SECRET=your_jwt_secret_key_here
```

### 3. 启动服务

**开发环境（推荐）：**

```bash
# 终端1 - 启动后端
cd backend
npm run dev

# 终端2 - 启动前端
cd frontend
npm run dev
```

访问：http://localhost:5173

**生产模式测试：**

```bash
# 构建前端
cd frontend
npm run build

# 启动后端（会同时提供前端静态文件）
cd ../backend
npm start
```

访问：http://localhost:3000

## 部署到 Render

### 快速部署步骤

1. **推送代码到 GitHub**

```bash
# 在项目根目录
git init
git add .
git commit -m "完整功能实现"

# 在 GitHub 创建仓库后
git remote add origin https://github.com/你的用户名/仓库名.git
git push -u origin main
```

2. **在 Render 创建服务**

- 访问 https://render.com/
- 点击 "New +" → "Web Service"
- 连接 GitHub 仓库
- 配置：
  - Build Command: `cd frontend && npm install && npm run build && cd ../backend && npm install`
  - Start Command: `cd backend && node server.js`
  - 添加环境变量（PORT, MONGODB_URI, JWT_SECRET）

3. **等待部署完成**

部署通常需要 5-10 分钟。

4. **初始化塔罗牌数据**

部署成功后，通过 Render Shell 或 API 初始化塔罗牌数据。

详细步骤见 `RENDER_DEPLOYMENT.md`

## 常见问题

### Q1: MongoDB 连接失败

**A:**
- 检查 MongoDB Atlas IP 白名单
- 确认连接字符串正确
- 尝试使用 `0.0.0.0/0` 允许所有 IP

### Q2: 前端无法访问后端 API

**A:**
- 开发环境：确保前端代理配置正确（vite.config.js）
- 生产环境：确保使用相对路径 `/api`

### Q3: 塔罗牌图片无法显示

**A:**
- 确认图片在 `D:\代课匹配\塔罗牌` 目录
- 确认 server.js 中配置了静态文件服务
- 检查图片路径：`/tarot-images/图片名.png`

### Q4: 部署后首次访问很慢

**A:**
- Render 免费套餐会在 15 分钟无活动后休眠
- 首次访问需要唤醒服务（约 30 秒）
- 考虑升级到付费套餐获得 24/7 运行

### Q5: 如何创建管理员账号

**A:**
1. 注册普通账号
2. 在 MongoDB 中手动修改该用户的 role 为 'admin'
3. 或在注册时临时修改代码允许注册管理员
