# 架构对比

## 之前的架构（有问题）

```
┌─────────────────────────────────────────────────────────┐
│                    Render 平台                           │
│                                                          │
│  ┌──────────────────┐         ┌──────────────────┐     │
│  │  前端静态站点     │         │   后端 API 服务   │     │
│  │  daike-frontend  │         │   daike-backend   │     │
│  │                  │         │                   │     │
│  │  URL: xxx-1.com  │ ◄─────► │  URL: xxx.com     │     │
│  │                  │  CORS!  │                   │     │
│  │  无法处理 SPA     │         │  只有 API 路由    │     │
│  │  刷新 → 404      │         │                   │     │
│  └──────────────────┘         └──────────────────┘     │
│                                                          │
└─────────────────────────────────────────────────────────┘

问题：
❌ 跨域请求（CORS 配置复杂）
❌ 静态站点无法处理 SPA 路由（刷新 404）
❌ 两个服务，配置复杂
❌ 需要维护两个 URL
```

## 现在的架构（已修复）

```
┌─────────────────────────────────────────────────────────┐
│                    Render 平台                           │
│                                                          │
│  ┌──────────────────────────────────────────────────┐  │
│  │         统一服务 (daike)                          │  │
│  │         Express 服务器                            │  │
│  │                                                   │  │
│  │  ┌─────────────────────────────────────────┐    │  │
│  │  │  路由处理顺序：                          │    │  │
│  │  │                                          │    │  │
│  │  │  1. /api/auth      → 认证 API           │    │  │
│  │  │  2. /api/availability → 空闲时间 API    │    │  │
│  │  │  3. /api/match     → 匹配 API           │    │  │
│  │  │  4. /*             → 静态文件            │    │  │
│  │  │  5. /*             → index.html (SPA)    │    │  │
│  │  │                                          │    │  │
│  │  └─────────────────────────────────────────┘    │  │
│  │                                                   │  │
│  │  URL: https://daike.onrender.com                 │  │
│  │                                                   │  │
│  └──────────────────────────────────────────────────┘  │
│                                                          │
└─────────────────────────────────────────────────────────┘

优势：
✓ 无跨域问题（同源）
✓ 完美支持 SPA 路由
✓ 单一服务，配置简单
✓ 只需一个 URL
✓ 更快的响应速度
```

## 请求流程

### API 请求
```
浏览器 → /api/auth/login
         ↓
    Express 路由匹配
         ↓
    authRoutes 处理
         ↓
    返回 JSON 响应
```

### 页面请求
```
浏览器 → /login
         ↓
    Express 路由匹配（无 API 匹配）
         ↓
    静态文件中间件（无匹配）
         ↓
    catch-all 路由
         ↓
    返回 index.html
         ↓
    Vue Router 接管
         ↓
    显示 Login 组件
```

### 静态资源请求
```
浏览器 → /assets/index-abc123.js
         ↓
    Express 路由匹配（无 API 匹配）
         ↓
    静态文件中间件
         ↓
    返回 JS 文件
```

## 构建流程

```
Render 构建阶段：
1. cd frontend
2. npm install          ← 安装前端依赖
3. npm run build        ← 构建前端（生成 dist/）
4. cd ../backend
5. npm install          ← 安装后端依赖

Render 运行阶段：
1. cd backend
2. npm start            ← 启动 Express 服务器
3. 服务器读取 ../frontend/dist 目录
4. 提供 API 和静态文件服务
```
